Adds an external hostpath volume to the deployment definition.

Adds an external hostpath volume to the deployment definition.
Output is a zip file printed to stdout.

Options:
    --db-hostpath='/var/lib/stackrox-central':
        Path on the host.

    --db-node-selector-key:
        Node selector key (e.g. kubernetes.io/hostname).

    --db-node-selector-value:
        Node selector value.

Global Options:
    --backup-bundle:
        Path to the backup bundle from which to restore keys and certificates.

    --ca:
        Path to a custom CA certificate to use (PEM format). Alternatively pass 
        the file path using the ROX_CA_CERT_FILE environment variable.

    --central-db-image:
        The central-db image to use (if unset, a default will be used according 
        to --image-defaults).

    --declarative-config-config-maps=[]:
        List of config maps to add as declarative configuration mounts in 
        central.

    --declarative-config-secrets=[]:
        List of secrets to add as declarative configuration mounts in central.

    --default-tls-cert:
        PEM cert bundle file.

    --default-tls-key:
        PEM private key file.

    --direct-grpc=false:
        Use direct gRPC (advanced; only use if you encounter connection issues).
        Alternately, enable by setting the ROX_DIRECT_GRPC_CLIENT environment 
        variable to true.

    --disable-admin-password=false:
        Disable the administrator password (only use this if you have already 
        configured an IdP for your instance).

    --enable-pod-security-policies=false:
        Create PodSecurityPolicy resources (for pre-v1.25 Kubernetes).

    --enable-telemetry=true:
        Whether to enable telemetry.

    -e, --endpoint='localhost:8443':
        Endpoint for service to contact. Alternatively, set the endpoint via the
        ROX_ENDPOINT environment variable.

    --force-http1=false:
        Always use HTTP/1 for all connections (advanced; only use if you 
        encounter connection issues). Alternatively, enable by setting the 
        ROX_CLIENT_FORCE_HTTP1 environment variable to true.

    -h, --help=false:
        Help.

    --image-defaults='rhacs':
        Default container images settings (rhacs, opensource); it controls 
        repositories from where to download the images, image names and tags 
        format.

    --insecure=false:
        Enable insecure connection options (DANGEROUS; USE WITH CAUTION). 
        Alternatively, enable insecure connection options by setting the 
        ROX_INSECURE_CLIENT environment variable to true.

    --insecure-skip-tls-verify=false:
        Skip TLS certificate validation. Alternatively, disable TLS certivicate 
        validation by setting the ROX_INSECURE_CLIENT_SKIP_TLS_VERIFY 
        environment variable to true.

    --istio-support:
        Generate deployment files supporting the given Istio version. Valid 
        versions: 1.0, 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7.

    --lb-type=none:
        The method of exposing Central (route, lb, np, none).

    -i, --main-image:
        The main image to use (if unset, a default will be used according to 
        --image-defaults).

    --no-color=false:
        Disable color output. Alternately disable the color output by setting 
        the ROX_NO_COLOR environment variable.

    --offline=false:
        Whether to run StackRox in offline mode, which avoids reaching out to 
        the Internet.

    --openshift-monitoring=auto:
        Integration with OpenShift 4 monitoring.

    --openshift-version=0:
        The OpenShift major version (3 or 4) to deploy on.

    --output-dir:
        The directory to output the deployment bundle to.

    --output-format=kubectl:
        The deployment tool to use (kubectl, helm, helm-values).

    -p, --password:
        Administrator password (default: autogenerated).

    --plaintext=false:
        Use a plaintext (unencrypted) connection; only works in conjunction with
        --insecure. Alternatively can be enabled by setting the ROX_PLAINTEXT 
        environment variable to true.

    --plaintext-endpoints:
        The ports or endpoints to use for plaintext (unencrypted) exposure; 
        comma-separated list.

    --scanner-db-image:
        The scanner-db image to use (if unset, a default will be used according 
        to --image-defaults).

    --scanner-image:
        The scanner image to use (if unset, a default will be used according to 
        --image-defaults).

    --scanner-v4-db-image:
        The scanner-v4-db image to use (if unset, a default will be used 
        according to --image-defaults).

    --scanner-v4-image:
        The scanner-v4 image to use (if unset, a default will be used according 
        to --image-defaults).

    -s, --server-name:
        TLS ServerName to use for SNI (if empty, derived from endpoint). 
        Alternately, set the server name via the ROX_SERVER_NAME environment 
        variable.

    --token-file:
        Use the API token in the provided file to authenticate. Alternatively, 
        set the path via the ROX_API_TOKEN_FILE environment variable or set the 
        token via the ROX_API_TOKEN environment variable.

    --use-current-k8s-context=false:
        Use the current kubeconfig context to connect to the central service via
        port-forwarding. Alternatively, set ROX_USE_KUBECONTEXT environment 
        variable to true.

Usage:
  roxctl central generate openshift hostpath [flags]
