# Image Registry Integration

Unified abstraction for connecting to container registries, retrieving metadata, and enabling vulnerability scanning across 11 registry types.

**Primary Packages**: `pkg/registries`, `central/imageintegration`

## What It Does

StackRox integrates with container registries to scan images and enforce policies. The system provides:

- Unified API across cloud-native (GCR, ECR, ACR), enterprise (Artifactory, Nexus, Quay), and Docker V2 registries
- Automatic token refresh for cloud providers
- Multi-architecture image support
- Tag listing with pagination
- Metadata retrieval (manifests, layers, Dockerfile instructions)
- Mirror/proxy registry support
- Delegated scanning where Sensor performs registry access

## Architecture

### Registry Interface

The `pkg/registries/types/types.go` defines core interfaces. Registry interface provides `Match()` for image hostname/repo matching, `Metadata()` for manifest/layer retrieval, `Test()` for connectivity validation, `Config()` for scanner credentials, and `ListTags()` for repository enumeration.

ImageRegistry extends Registry with `DataSource()` returning integration ID/name and `Source()` providing full integration config.

### Supported Types

**Cloud Providers**:

| Registry | Type | Auth | Token Refresh | Location |
|----------|------|------|---------------|----------|
| AWS ECR | ecr | IAM/assume-role/IRSA | 10 min before expiry | `pkg/registries/ecr/` |
| Google GCR/GAR | google/artifactregistry | Service Account/WIF | OAuth2 automatic | `pkg/registries/google/` |
| Azure ACR | azure | Username-password/WIF | AAD token exchange | `pkg/registries/azure/` |

**Enterprise**:

| Registry | Type | Implementation | Location |
|----------|------|----------------|----------|
| Quay | quay | OAuth token/robot creds | `pkg/registries/quay/` |
| Artifactory | artifactory | Delegates to docker | `pkg/registries/artifactory/` |
| Nexus | nexus | Broken digest endpoint workaround | `pkg/registries/nexus/` |

**Other**: Docker Hub/generic V2 (base for most types), GitHub Container Registry (delegates to docker), IBM Cloud Registry (IAM API key), Red Hat Registry (broken digest endpoint workaround).

### Factory Pattern

The `pkg/registries/factory_impl.go` maintains two creator maps: with and without repo list support. Selection logic uses creators without repo list if integration is user-created (not autogenerated), DisableRegistryRepoList env var is set, or integration has a Source (namespaced pull secret).

Each registry type provides two creator functions in its package: standard creator with repo list for precise matching, and creator without repo list for performance or user integrations.

### Matching and Priority

The `pkg/registries/set_impl.go` Set interface manages registry collections. Priority order: registries with username/password credentials first, non-autogenerated over autogenerated, exact match by data source ID if image has one.

Matching algorithm: check data source ID for direct lookup, otherwise iterate sorted registries and return first match. Repository list matching uses hostname check followed by optional repo list containment check if enabled.

Repository lists lazily load on first match, refresh every 10 minutes, use expensive `/v2/_catalog` API, and can disable via env var or creator selection.

### Metadata Retrieval

Supported manifest formats: Docker Schema V1 (deprecated), Docker Schema V2, Docker Manifest List V2 (multi-arch), OCI Image Manifest V1, OCI Image Index V1.

Multi-architecture handling in `pkg/registries/docker/metadata_v2.go`: use single manifest if only one exists, prefer linux/$GOARCH, fallback to linux/amd64.

Metadata structure includes V1Metadata (layers, Dockerfile history, labels, volumes, user, command, entrypoint, created time) and V2Metadata (manifest digest).

## Authentication

### Token Refresh

**ECR** (`pkg/registries/ecr/transport.go`): awsTransport maintains config, client, expiresAt, and mutex. Refresh occurs 10 minutes before expiry via GetAuthorizationToken API call.

**GCR/GAR** (`pkg/registries/google/transport.go`): googleTransport uses OAuth2 TokenSource with automatic refresh.

**ACR** (`pkg/registries/azure/transport.go`): azureTransport gets AAD access token via Azure credential chain, then exchanges for ACR refresh token.

### Tag Listing

Added in ROX-31821 for base image watching and vulnerability scanning triggers. Implementation in `pkg/registries/docker/docker.go` uses go-containerregistry for proper pagination and auth. Features include automatic pagination, bearer token authentication, same TLS/timeout config as other operations, and per-request timeouts.

## Auto-Generated Integrations

**GlobalPullSecret**: Sourced from OpenShift global pull secret in `openshift-config/pull-secret` (ROX-23735). Prefix: `GlobalPullSecret`.

**Per-Namespace Pull Secrets**: Created from service account imagePullSecrets. Stored with namespace + secret name in Source field for namespace-scoped matching. Prefix: `PullSecret`.

**No-Auth Registries**: Autogenerated for public registries discovered in deployments. Prefix: `NoAuth`.

## Metrics

The `pkg/registries/types/metrics.go` defines Prometheus metrics:

- `stackrox_<subsystem>_registry_client_requests_total`: Counter with code/method/type labels
- `stackrox_<subsystem>_registry_client_error_timeouts_total`: Counter with type label
- `stackrox_<subsystem>_registry_client_request_duration_seconds`: Histogram with type label, exponential buckets 0.1s to 20s

Instrumentation: `transport = cfg.MetricsHandler.RoundTripper(transport, cfg.RegistryType)`

## Configuration

**Environment Variables**:
- `ROX_DISABLE_REGISTRY_REPO_LIST`: Force all registries to use CreatorWithoutRepoList
- `ROX_REGISTRY_CLIENT_TIMEOUT`: HTTP client timeout
- `ROX_REGISTRY_DIALER_TIMEOUT`: TCP connection timeout
- `ROX_REGISTRY_RESPONSE_TIMEOUT`: HTTP response header timeout

**Delegated Scanning**: Sensor performs registry operations on behalf of Central, reducing Central's network egress, working with inaccessible registries, and leveraging cluster service accounts.

## Known Limitations

- Some registries (Red Hat, Nexus) have broken digest endpoints requiring workarounds
- Large registries (thousands of repos) can timeout on `/v2/_catalog`
- Cloud tokens refresh on-demand except ECR's proactive refresh
- Multi-arch platform hardcoded to prefer linux/amd64 or linux/$GOARCH
- No built-in Docker Hub rate limiting retry or backoff
